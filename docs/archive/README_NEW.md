## 这是什么

你刷到一条"美伊局势升级"的抖音，评论区吵翻了，但你连谁打了谁都没搞清楚。

你看到一条明星塌房的爆料，各路营销号说法不一，到底谁在说谎？

**PulseGraph 就是为这种时刻而生的。**

丢一条链接进来——新闻也好、抖音也好——它会：

1. 🎙️ **自动听完视频在说什么**（ASR 语音转文字，不用你手动整理）
2. 🔍 **主动去找更多相关报道**（不是你喂什么它吃什么，它自己会去"扒"）
3. 🕸️ **画出一张人物关系图谱**（谁是谁的盟友、谁和谁撕破脸、一目了然）
4. ⏳ **梳理事件时间线**（从导火索到最新进展，因果链条清清楚楚）
5. ⚠️ **标出哪些信息在"打架"**（A 媒体说复合了，B 媒体说没有——红色警告给你标出来）
6. 💬 **你还能直接问它**（"这件事的起因是什么？""他俩到底什么关系？"）

简单说：**它是你的私人情报分析师，专治信息碎片化。**

## 它能干什么

### 场景一：国际新闻（谁在打谁？为什么打？）

> 输入：一条关于美伊冲突的抖音视频链接
>
> 输出：美国、伊朗、以色列的关系网 + 制裁→报复→升级的因果链 + "路透社说X，抖音博主说Y"的冲突标注

### 场景二：明星八卦（到底什么瓜？）

> 输入：一篇爆料文章链接
>
> 输出：当事人关系图（前任、经纪人、同公司艺人）+ 事件发展时间线 + 各方回应汇总

### 场景三：影视剧情（人物关系太复杂记不住？）

> 输入：几篇剧评/原著片段
>
> 输出：角色关系导图 + 剧情因果链

## 凭什么它能做到

不是套个 ChatGPT 壳子就完事了。底下是一套完整的 **Agentic RAG + 动态知识图谱** 引擎：

| 你看到的 | 底下在跑的 |
|----------|-----------|
| "自动听视频" | yt-dlp 下载 → ffmpeg 提取音频 → Whisper ASR 转写 |
| "主动找报道" | Agent 自动生成搜索策略 → 多源检索 → 相关性打分 → 信息饱和度判断自动停止 |
| "画关系图" | LLM 实体关系抽取 → 实体消歧 → Neo4j 图谱构建 → D3.js 力导向可视化 |
| "标出矛盾" | 时序重叠检测 + 逻辑矛盾检测 + 多源分歧检测 |
| "能问能答" | OpenAI Function Calling → 图查询/向量检索/因果推理 多工具协作 → 置信度验证 |

## 技术架构

```
┌─ 输入层 ──────────────────────────────────────────────┐
│  新闻 URL │ 抖音链接(ASR转写) │ 文本/PDF 上传         │
└──────────────────────────┬────────────────────────────┘
                           ▼
┌─ 主动补全 ───────────────────────────────────────────┐
│  种子分析 → 搜索策略生成 → 多轮检索 → 饱和度停止     │
└──────────────────────────┬───────────────────────────┘
                           ▼
┌─ 知识图谱演化 ───────────────────────────────────────┐
│  实体抽取 → 消歧去重 → 冲突检测 → 时序版本化入图     │
└──────────────────────────┬───────────────────────────┘
                           ▼
┌─ Agent 推理 ─────────────────────────────────────────┐
│  状态机编排 → Function Calling → 多工具协作 → 验证   │
└──────────────────────────┬───────────────────────────┘
                           ▼
┌─ 展示层 ─────────────────────────────────────────────┐
│  关系图谱 │ 事件时间线 │ 因果链路 │ 冲突面板 │ 问答  │
└──────────────────────────────────────────────────────┘
```

## 技术栈

| 干什么的 | 用什么 |
|----------|--------|
| 后端 | Python 3.11 + FastAPI |
| Agent 推理 | OpenAI Function Calling + 自研状态机编排 |
| 知识图谱 | Neo4j 5.x |
| 向量检索 | Qdrant |
| 语音转写 | Whisper / DashScope ASR |
| 视频获取 | yt-dlp + ffmpeg |
| 异步任务 | Celery + Redis |
| 前端 | React 18 + TypeScript + D3.js |
| 部署 | Docker Compose |

## 快速开始

```bash
# 1. 克隆
git clone https://github.com/liu66-qing/PulseGraph.git
cd PulseGraph

# 2. 启动依赖（Neo4j + Redis + Qdrant）
docker-compose up -d

# 3. 配置
cp .env.example .env
# 填入 OPENAI_API_KEY

# 4. 安装 & 启动
pip install -e ".[dev]"
make run

# 5. 前端
cd frontend && npm install && npm run dev
```

打开 http://localhost:5173，丢个链接进去试试。

## 和"普通 RAG 项目"有什么区别

| | 普通 RAG | PulseGraph |
|---|---|---|
| 输入 | 你喂文档 | 你丢一条链接，它自己去找更多 |
| 知识表示 | 文档碎片 | 结构化知识图谱 + 时序版本 |
| 检索 | 向量相似度 | 图遍历 + 向量 + 关键词 三路融合 |
| 推理 | 检索→生成（固定管道） | Agent 动态规划（会回头、会追问、会验证） |
| 冲突处理 | 没有 | 三类冲突自动检测 + 来源可靠度评分 |
| 输出 | 一段文字 | 关系图谱 + 时间线 + 因果链 + 文字回答 |
